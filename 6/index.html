Pertemuan kali ini membahas tentang class diagram
1. Class diagram untuk menggambarkan hubungan antar tebel, tabel ini digambarkan untuk classnya digambarkan melalui class diagram tepatnya pada struktur tabel
2. Class diagram terdiri dari : nama class, atribut (lebih dari satu), dan juga method
3. Nama class yaitu nama tabel nya, nama class ini tidak bisa menggunakan spasi
4. Atribut yaitu isi field tipe data atau length dan batasnya, ditentukan juga sebagai keynya.
5. Method yaitu opersi untuk kaitan dengan hubungan basis data class/tabel seperti select/baca, insert/tambah/simpan, update/ubah,delete/hapus, count/hitungjumlah, auto_numbering/nomor otomatis/kode otomoatis, dll, maka ambil detailnya saja.
6. Class diagram itu bisa juga dibilang gabungan dari representasi dari ERD dan LRS
7. Stereotype fungsinya menentukan dan menandakan attribute yang menjadi pk(primary key), fk (foregn key).
8. Multiplicity fungsinya menentukan tingkat hubungan /kardnalitas anatar class melalui association (garis hungan antar class)
9. Di pertemuan 6 ini pembahasannya menggunakan software ASTAH COMMUNITY 6
10. Dipembahasan sebelumnya kita sudah membuat diagram rancangan layar sebagai contohnya
11. Memahami penjelasan diagram layar konsumen
